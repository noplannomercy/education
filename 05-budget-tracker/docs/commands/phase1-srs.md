# Phase 1: SRS 작성

## 언제 사용
요구사항 확정 후. **references는 건드리지 않는다.**

---

## 프롬프트

```
docs/SRS.md 작성해줘.

프로젝트: [프로젝트명]
주요 기능:
- [기능1]
- [기능2]
- [기능3]

다음 항목 포함:
- 기능 요구사항 (요구사항 ID 포함, 예: LEAD-01)
- 데이터 모델 개요 (엔티티 관계)
- 상태 정의 (있는 경우)
- 화면 목록 (URL 포함)
- 개발 범위 외
```

---

## 완료 기준
- [ ] `docs/SRS.md` 존재
- [ ] 기능 요구사항에 ID 부여됨
- [ ] 엔티티 관계 + 화면 목록 + URL 명시

---

## 주의

SRS는 **무엇을 만들지** 정의한다.
**어떻게 만들지**는 Phase 0에서 수립된 references가 정의한다.
SRS 내용이 references 패턴과 충돌하면 → references 변경 절차(phase0 참고) 먼저.
