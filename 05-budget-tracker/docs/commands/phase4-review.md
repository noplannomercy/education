# Phase 4: 결과 리뷰

## 언제 사용
주요 기능 구현 완료 후, 또는 Chunk 단위로 수시 실행

## 프롬프트 (코드 품질 리뷰)
```
다음 파일들 코드 리뷰해줘.

리뷰 기준:
- 구현계획(docs/superpowers/plans/)의 모든 Chunk가 구현됐는가
- docs/SRS.md 의 기능 요구사항과 실제 코드가 일치하는가
- [프로젝트 스킬] 컨벤션 준수 여부 (레이어 분리, 네이밍, 에러 처리)
- [프레임워크 스킬] 컨벤션 준수 여부
- TypeScript strict 준수 (any 금지, 반환 타입 명시)
- 인증 처리 누락 없는지
- 비즈니스 로직 엣지케이스

리뷰 대상:
- [서비스 레이어 경로]
- [API 레이어 경로]
- [컴포넌트 경로]
- [페이지 경로]

출력: Critical / Important / Minor / 잘된 점 / 종합 평가
```

> **리뷰 깊이 기준 (LLM 판단)**
> - 전체 기준 적용: 비즈니스 로직, 인증, DB 변경이 포함된 경우
> - 컨벤션·타입만: UI 컴포넌트, 스타일, 레이아웃만 변경된 경우

## 프롬프트 (브라우저 동작 검증)
```
e2e 테스트 해줘.
```

> 워크플로우 + LLM 판단 기준 → `e2e-testing.md` 참고

## 프롬프트 (Critical 이슈 수정)
```
코드 리뷰 결과 Critical 이슈 수정해줘.
[프로젝트 스킬] 참고해서 기존 패턴 유지하면서 수정.
수정 후 [빌드 명령] 확인.
```
