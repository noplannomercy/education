### 왜 Claude Code인가 (3분)

> 📌 **[슬라이드 4]** 왜 Claude Code인가

"그럼 왜 Claude Code를 쓰는가. Codex, OpenCode도 있는데 왜 Claude Code냐고요.

개발자 입장에서 가장 중요한 건 2번이에요. 내 환경 그대로 쓴다는 거.

Replit에서 만들면 Replit에 올라가요. Lovable에서 만들면 Lovable 방식으로 배포돼요. Claude Code로 만들면 내 로컬에 코드가 있고, 내가 원하는 데 배포할 수 있어요.

개발자분들이니까 이미 Git 쓰시고, 로컬 환경 있으시고, 배포 파이프라인 있으신 분들 계실 텐데, 그 위에 그냥 얹으면 돼요."

"Codex CLI, OpenCode도 좋아요. 방향은 같아요. 오늘은 Claude Code로 진행하는 거고, 원리를 익히면 다른 도구도 비슷하게 쓸 수 있어요."

**마인드셋 ①: 코드를 읽지 마라, 결과를 읽어라**

"AI가 코드 짜는 동안 여기 계신 개발자분들 중에 코드 보시는 분들 계실 거예요. '이거 맞나? 이 로직이...' 하면서요.

그러면 바이브코딩을 못 쓰는 겁니다.

AI가 만든 코드는 AI한테 맡겨요. 여러분이 판단해야 할 건 딱 하나예요. '내가 원하는 대로 동작하는가.'

브라우저 켜고, 직접 써보고, 원하는 대로 안 되면 말로 요청해요. 이게 전부입니다."

**마인드셋 ②: 완벽한 프롬프트 없다, 일단 던져라**

"처음에 프롬프트 어떻게 써야 하나 고민하는 분들이 많아요. '이렇게 쓰면 제대로 나올까?' 하면서 5분 동안 프롬프트 다듬는 거요.

그냥 던지세요.

'Todo 앱 만들어줘' 이거면 충분해요. 결과 보고 마음에 안 들면 그때 수정 요청하면 됩니다. 첫 프롬프트가 완벽할 필요 없어요. 바이브코딩은 대화예요. 한 번에 끝내는 게 아니라 주고받으면서 만들어가는 겁니다."

**마인드셋 ③: AI는 기억이 없다, 컨텍스트가 품질이다**

"이게 가장 중요한 거예요.

AI는 지금 이 대화창에 있는 것만 알아요. 이 대화가 끝나면 다 잊어요. 그래서 AI한테 뭘 알려주느냐가 결과 품질을 결정합니다.

지금 만든 Todo 앱은 4분 만에 만들었는데, 기능이 간단하니까 AI가 혼자 결정해도 됐어요. 근데 복잡한 앱을 만들면? 기능이 10개, 20개가 되면? AI가 매번 다르게 결정해요. 이번엔 이 패턴, 다음엔 저 패턴. 나중에 기능 추가하면 충돌나요.

이 문제를 해결하는 게 오늘 3막에서 다룰 스펙주도개발입니다. 지금은 이런 게 있다는 것만 알아두세요."

## 2막: 품질 레벨업 — Superpowers (11:30~13:00)

### ① Superpowers란 (11:30~11:50)

> 📌 **[슬라이드 11]** Superpowers — 자동화된 개발 프로세스

"2막에서는 Superpowers라는 도구를 소개해드릴 건데요.

먼저 문제부터 얘기할게요.

1막에서 만든 Todo 앱, 괜찮았죠? 근데 거기서 기능을 계속 추가해보세요. '로그인 기능 추가해줘', '카테고리 분류 추가해줘', '통계 화면 추가해줘'... 한 10개 요청하면 어떻게 돼요?

AI가 매번 달라요. 첫 번째 기능은 이 패턴으로 짰는데 다섯 번째 기능은 저 패턴으로 짜요. 변수명도 다르고, 에러 처리 방식도 다르고. 나중에 뭔가 고치려고 하면 코드가 얽혀있어서 손을 못 대요.

왜 그러냐고요? AI는 대화가 길어질수록 앞에서 어떻게 짰는지 점점 잊어요. 컨텍스트가 오염되는 거예요.

이게 장난감 앱이면 괜찮아요. 근데 실제로 쓸 앱을 만들면? 나중에 다 갈아엎게 돼요.

Superpowers는 이 문제를 구조적으로 해결해요.

슬라이드 보시면 5단계가 있어요. 브레인스토밍 → 설계 → 계획 → 실행 → 테스트.

핵심은 두 가지예요. 첫 번째, 코드 짜기 전에 설계를 먼저 문서로 고정해요. AI가 코드 짤 때 이 문서를 기준으로 짜니까 일관성이 유지돼요. 두 번째, 기능마다 AI를 새로 시작해요. 앞 기능의 컨텍스트가 다음 기능에 영향을 안 줘요.

결과가 어떻게 다른지는 제가 미리 만들어온 걸 보여드릴게요."

"SRS가 나오면 그 다음은 구현계획이에요.

```
├── database.md   → DB 스키마, 쿼리 패턴

└── conventions.md  → 코드 스타일, 네이밍 규칙
```

"이 4개 파일이 있으면 AI가 어떤 기능을 만들든 여기에 쓰인 패턴을 따라요. 신입 개발자한테 코딩 가이드를 주는 것과 같아요.

이게 없으면 AI가 매번 다르게 결정해요. 이게 있으면 처음부터 끝까지 같은 패턴이에요."
