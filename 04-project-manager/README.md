# 2.2.test-project-manager

바이브코딩 워크샵 시연용 프로젝트 관리 앱.
기능 구현 후 피처 추가 흐름을 보여주는 예시.

---

## 시연 포인트

**"완성된 앱에 기능을 추가할 때도 Superpowers가 자동 발동된다"**

---

## 단계별 흐름

### 1단계 — 프로젝트 매니저 구현

```
HTML CSS JS 로컬스토리지 기반 프로젝트 관리 앱 만들어줘.
기능은 이것만:
- 프로젝트 CRUD
- 프로젝트별 태스크 CRUD
- 태스크 상태 (할일/진행중/완료)

PRD는 핵심만 docs/PRD.md로 작성하고 바로 구현해줘.
```

→ Superpowers 자동 발동 → `docs/plans/2026-03-07-project-manager.md` 생성 → 구현

### 2단계 — 버그 확인

```
완성된 앱의 모든 기능이 정상 동작하는지
확인하고 동작 안 하는 것 있으면 바로 수정해줘.
```

### 3단계 — 칸반보드 피처 추가

```
현재 앱을 칸반보드 형태로 변경해줘.
태스크를 상태별 3개 컬럼으로 표시:
- 할일 | 진행중 | 완료
카드 드래그앤드롭으로 컬럼 간 이동 가능.
```

→ Superpowers 다시 발동 → `docs/plans/2026-03-07-kanban.md` 신규 생성 → 칸반 구현

---

## 기능

- 프로젝트 생성 / 수정 / 삭제
- 프로젝트별 태스크 CRUD
- 칸반보드 (할일 / 진행중 / 완료 3컬럼)
- 태스크 카드 드래그앤드롭으로 상태 변경
- LocalStorage 영속 저장
